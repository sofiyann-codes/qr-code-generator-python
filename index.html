Hosting this for free

This page is a single static HTML/CSS/JS bundle — no server, no build step, no database. That means it's free to host anywhere that serves static files. Two solid options:

GitHub Pages recommended

Hosted straight from your existing repo, on a URL like sofiyann-codes.github.io/qr-code-generator-python.

  1. Add index.html, style.css, and script.js to the repo root.
  2. Commit and push to the main branch.
  3. On GitHub: Settings → Pages → Build and deployment.
  4. Source: Deploy from a branch → Branch: main, folder / (root) → Save.
  5. Wait ~1 minute, then visit the URL GitHub shows on that same page.

Netlify drag & drop

No GitHub steps required if you just want a live link fast.

  1. Go to app.netlify.com/drop.
  2. Drag the folder containing index.html, style.css, script.js onto the page.
  3. Netlify gives you a live *.netlify.app URL instantly.
  4. Optional: claim a custom subdomain in Site settings → Domain management.

Both are genuinely free for a static site like this one — no card required, no traffic cap you'd realistically hit.

Good to know

Does any of this touch a server?

No. Every code is built locally in your browser with JavaScript. Nothing you type is uploaded, logged, or stored anywhere.

Why does my Wi-Fi QR code show the password as plain text?

That's inherent to the Wi-Fi QR format, not this tool — anyone who scans the code can see the network's credentials. Only share Wi-Fi codes with people you'd trust with the password directly, and avoid posting them somewhere public.

My scanner says the code is unreadable. What happened?

Usually low contrast between the code and background, a size too small for the amount of data, or a low error-correction setting. Raise the contrast, bump the size up, or switch error correction to Q or H.

Can I use this for a printed poster?

Yes — generate at a larger size (400px+) before downloading so it stays sharp at print resolution, and always test-scan the printed result before mass-printing.
